Using stylesheets with AntMenus and AntForms

Starting with version 1.3, stylesheets can be used to add style to menus and forms. a stylesheet is basically a properties file with keys and values that map to widget type and style properties.

Given the widgets, the following properties can be set:

  • background.color: hexadecimal (ex: #FFFFFF) formatted color to use for background.
  • color: hexadecimal formatted color to use for text.
  • font.family: os-supported font family (ex: Courier, Roman, etc.)
  • font.size: font size in pixels
  • font.weight: weight of the font, among the following values: "plain|bold|italic|bold-italic"
  • border.style: set to "solid" to overload default border
  • border.width: width of the border in pixels
  • border.color: hexadecimal formatted color to use for border.

The following widgets can be styled:

  • textProperty
  • banner (used at top of form windows)
  • multiLineTextProperty
  • scrollPanes (basically sets scrollpane behaviour for multilineTextProperty widgets)
  • button (for bottom panel form button)
  • link
  • label (maps to left-column labels)
  • message (maps to label tags)
  • checkbox
  • selectionProperty

Not all properties can be styled. The mapping was established with a view to make basic Swing look-and-feels look ok, so certain border properties are not active and the checkbox only uses the background.color attribute.

To use a stylesheet, write out the properties in the following format (see the style.test stylesheet included in the distribution:

<widget>.<property>=<value>

for example:

textProperty.background.color=#FFFFFF

Save the properties file and copy the file path of the stylesheet to the stylesheet attribute of either the antmenu or antform tasks.
