Smooth Look And Feel

Author

Marcel Offermans - original idea and code.
James Shiell - bug fixes and 0.3 onwards. B. K. Oxley (binkley) - bug fixes and 1.1 onwards.

Description

The SmoothMetal Look and Feel started out as a pure replacement for the Java Metal Look and Feel, with the expressed aim of designing a replacement for this lacklusture effort by Sun. With Sun's expressed aim of redesigning Metal for the Tiger release of Java (1.5) recent work has been targetted at bugfixes and enabling anti-aliasing for the other Java Look and Feels.

Usage

smooth.SmoothLookAndFeelFactory.getSystemLookAndFeelClassName(); returns the appropriate look and feel classname for the current OS, or the standard system look and feel classname if the system look and feel is not Metal or Windows.

UIManager.setLookAndFeel("smooth.metal.SmoothLookAndFeel"); sets the look and feel to Smooth Metal.

UIManager.setLookAndFeel("smooth.windows.SmoothLookAndFeel"); sets the look and feel to Smooth Windows.

To use it with an existing, unmodified application, try java -Dswing.defaultlaf=smooth.windows.SmoothLookAndFeel <appname> (change the LnF classname as appropriate). Note that some applications may force a certain look and feel and thus essentially ignore this define.

If you use the TitledBorder class, you'll need to replace it with SmoothTitledBorder. Apart from this AA will be enabled automatically.

smooth.util.SmoothUtilities contains methods to allow toggling of anti-aliasing and fractional font metrics. Note that you'll need to force a repaint after calling these methods.

java -jar smoothmetal.jar or java -cp smoothmetal.jar smooth.demo.Main runs the demo frame, showing a number of components in the Smooth look and feels. Under Windows, just double click smoothmetal.jar.

Future Features

These are just suggestions - I will be making no further releases.

  • Support for the Motif Look and Feel.
  • Support for the GTK Look and Feel (JRE 1.4.2 and above).
  • Support for the Third Party Look and Feels (Such as Alloy).

ReleaseDatesNotes
1.1 07/02/2006 21/04/2005
  • Compilable with JDK 1.4.2.
  • Tested with JDK6 beta.
  • Smooth borders of internal frames, internal dialogs and color palettes for SmoothMetal.
  • Update demo app to demonstrate the new smoothings.
  • Show demo app with a JDK default border rather than the native platform border to demonstrate smooth frames better.
  • In demo app toggling anti-alias and font metrics rebuilds the UI from the top-level pane down, not just refreshing it. Smoothed fonts often have somewhat different metrics than unsmoothed, and the layout is imperfect otherwise.
  • Bugfix for JSpinner & FormattedTextFields (990889).
  • Bugfix for Windows LnF name (1186537).

  • Bugfix for Windows LnF icons (905623).

  • Now requires J2SDK 1.5.0B1 to build (runs fine under 1.4.2_03).
